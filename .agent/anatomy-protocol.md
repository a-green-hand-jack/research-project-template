# Anatomy Protocol

`ANATOMY.md` is a map, not a manual.

## Rules

- Root `ANATOMY.md` routes ownership and state surfaces.
- Nested anatomy belongs only in directories that are independent concepts.
- Structural edits update anatomy in the same change.
- Before moving, renaming, splitting, or deleting files, search all anatomy and
  index files for touched names.
- If an anatomy file grows beyond a concise map, split the underlying boundary
  or move tutorial content to `README.md`.

## Do Not

- Do not create placeholder anatomy for simple leaf directories.
- Do not turn anatomy into a full design document.
- Do not update code now and anatomy later.
