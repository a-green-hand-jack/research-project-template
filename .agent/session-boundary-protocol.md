# Session Boundary Protocol

Long research work should branch as a tree, not remain one giant chat.

## Files

- `memory/session-tree.md` records parent and child session branches.
- `memory/sessions/` stores handoff notes.
- `plans/` stores durable human-agent plans when discussion is needed.
- `memory/current-status.md` stores the next entry point for a fresh agent.

## Boundary Triggers

Open a child session, plan file, or worktree when:

- exploration becomes implementation
- implementation becomes verification or paper promotion
- failed hypotheses are polluting the current context
- independent tasks can be owned by separate branches
- a remote or expensive side effect needs a gate
- the task changes target venue, main claim, baseline, metric, or data split

Before compact, clear, or handoff, update the smallest state file that lets a
fresh session resume without trusting chat memory.
