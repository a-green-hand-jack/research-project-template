# Artifact Indexing

Use this skill after experiments, table generation, figure generation, dataset
preparation, checkpoint creation, or paper-claim promotion.

1. Read `.agent/artifact-policy.md`.
2. Locate the relevant run, dataset, checkpoint, table, figure, or document
   asset.
3. Update or propose the appropriate index under the active component.
4. Link claim-facing assets to commit, config, run id, metric source, artifact
   path, and inspection command.
5. Mark old assets active, superseded, archived, blocked, or unknown.
6. Do not delete or move large artifacts without explicit approval.

Output should answer where the asset is, how to inspect it, what claim depends
on it, and whether it is still active.
